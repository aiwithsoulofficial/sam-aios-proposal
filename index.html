Phase 1

Your AIOS Build Plan

Three phases, built in order of urgency. Phase 1 is the priority. Everything else follows once you have your time back.

1
Free Sam's Time
$5,000 - Delivered within 30 days
Goal: Give you back 10-20 hours per week within 30 days. Every deliverable in this phase is chosen because it attacks the biggest time drains directly.
  • 1
    Context OS
    Your business brain, installed in Claude. Every interaction knows your services, your team, your participants, your compliance requirements, and how you like things done. Institutional knowledge that never leaves, never forgets, and never needs to be re-explained.
    Technical spec: Claude Code CLI installed on Sam's machine via TeamViewer. CLAUDE.md configuration file containing business rules, NDIS compliance requirements, service types and pricing ($100.14-$193.99/hr), team roles (Sam, Leah, Christine, Monika, JC), participant data handling rules, and communication preferences. Persistent memory system using markdown files for participant profiles, provider contacts, document templates, and recurring deadlines. Folder structure mapped to Access Assist Co's SharePoint hierarchy. All data stored locally on Sam's machine - Australian data residency.
  • 2
    Email Triage Agent
    AI reads every incoming email, classifies it, extracts the action needed, and creates a Planner task automatically. Every morning, Telegram sends you a snapshot of the three things to handle before anything else. Your inbox stays in your inbox - you told us your brain needs it there - but it no longer controls your day.
    Technical spec: Microsoft Graph API (OAuth 2.0) connecting to Sam's Outlook mailbox. Scheduled launchd agent runs every 15 minutes - pulls new emails via Graph API, sends each to Claude for classification (NDIS / provider / participant / admin / urgent). Claude extracts action items and creates tasks in Microsoft Planner via Graph API with correct bucket, priority, and due date. Emails stay in inbox (not moved to folders). Morning briefing aggregates overnight triage results and delivers via Telegram Bot API at 7am ACST. Existing Power Automate flow (email flag to Planner) reviewed and either integrated or replaced. Sam's Anthropic API key used for all Claude calls.
  • 3
    Case Note Automation /case-note
    Record a call. Get NDIS-compliant case notes in two minutes. Not 30. No more five-step copy-paste loop. Works for your whole team, not just you. Sam, JC, Leah, Christine - everyone saves the same amount of time on every single visit.
    Technical spec: Audio recording (voice memo or Speak App export) sent to OpenAI Whisper API for transcription. Transcript passed to Claude with NDIS-compliant case note template (date, time, duration, charge item code, participant name, narrative). Claude formats into structured case notes matching Astalty's required fields. Output delivered as formatted text ready to paste into Astalty, or pushed directly via Astalty data bridge (Phase 2). Skill invoked via Claude Code CLI (/case-note) or Telegram voice note. All four coordinators can use independently.
  • 4
    Morning Briefing Agent /morning
    Daily Telegram message: what's urgent, what's due, quick wins first. Dopamine-friendly ordering so the hardest things don't paralyse the start of the day. You know exactly where to put your energy before you've had coffee.
    Technical spec: Scheduled launchd agent fires at 7am ACST daily. Pulls data from three sources: Microsoft Graph API (Outlook unread count + flagged emails, Planner tasks due today/overdue), email triage agent results (overnight classification), and local memory files (participant deadlines, plan review dates). Claude synthesises into a prioritised briefing - quick wins first (under 5 min tasks), then urgent items, then scheduled work. Delivered via Telegram Bot API as a formatted message. Includes clickable deep links to Outlook, Planner, and Astalty where relevant.
  • 5
    Command Centre
    One screen. Everything. Inbox status, tasks, case notes, invoices, team activity, audit prep. Click through to any system in your stack. No more six tabs. No more context switching. The full picture of your business, at a glance.
    Technical spec: Single-page HTML dashboard served locally or via Render. Pulls live data from: Microsoft Graph API (Outlook inbox count, Planner tasks, SharePoint files), Astalty (participant data, case notes, invoice status - via Playwright browser automation since no API exists), and Xero API (invoice reconciliation status, revenue tracking). Auto-refreshes every 5 minutes. Each item links directly to its source system. Separate views for Sam (full operational) and team (simplified). Built with vanilla HTML/CSS/JS - no framework dependencies.
  • 6
    Telegram Voice Command AIOS
    Driving to a home visit? Voice note to Telegram. "Draft an email to Jordan's plan manager about the OT referral." Done before you park. "Create a task for JC to upload Riley's consent forms by Friday." Done while you're still driving. Full hands-free delegation, no scraps of paper required.
    Technical spec: Custom Telegram bot (Bot API) listening for voice notes from Sam's account. Voice note received > downloaded > sent to OpenAI Whisper API for transcription > transcript sent to Claude for intent classification and routing. Supported intents: draft email (triggers /draft-email skill), create Planner task (Microsoft Graph API), log case note (triggers /case-note skill), query participant info (reads from memory/Astalty), delegate to team member (creates assigned Planner task). Claude confirms action via Telegram reply. All routing logic lives in a Python script running as a launchd agent on Sam's machine. Sam's Anthropic API key and Telegram bot token stored in local .env file.
  • 7
    Multi-Voice Email Drafting /draft-email
    Three voice profiles built from your actual writing: formal for NDIS correspondence and court documents, warm for participants and families, internal for the team. Drafts that sound like you, not like AI. You review and approve before anything sends - always.
    Technical spec: Three voice profile templates stored in Claude's memory system, each built from samples of Sam's actual sent emails (minimum 10 samples per profile). Profile 1: Formal - NDIS correspondence, DCP reports, court letters, plan manager communication. Profile 2: Warm - participant and family communication, referrer updates. Profile 3: Internal - team instructions, JC task delegation. Invoked via /draft-email skill in Claude Code or via Telegram voice command ("draft an email to X about Y, warm tone"). Claude generates draft using the matched profile. Draft delivered to Sam via Telegram for approval - thumbs up sends via Microsoft Graph API (Outlook send), thumbs down returns for revision. Human-in-the-loop on every send - no auto-sending.
Phase 2

Systemise the Admin

2
Systemise the Admin
$5,500 indicative - Months 3-4, scheduled when you're ready
Goal: Automate the remaining manual admin that eats your weekends. By the time you get here, Phase 1 has already given you breathing room. This phase gets your Sundays back.
  • 1
    NDIS Document Drafting /ndis-doc
    Upload a clinical report, select the template type, get a draft plan reassessment in minutes. Not hours. Built around NDIS Practice Standards so the output is compliant from the start.
    Technical spec: Claude Code skill (/ndis-doc) accepting document type (change of situation, plan reassessment, SC justification, court letter) and participant identifier. Pulls participant data from Astalty via data bridge. Accepts uploaded clinical reports (OT, speech, medical - PDF or Word) which Claude reads and extracts relevant findings. Pre-built NDIS-compliant Word templates stored locally. Claude generates draft content section by section, populating the template. Output as Word document for Sam's review before submission. Supports bulk report ingestion (upload multiple clinical reports, Claude cross-references all).
  • 2
    Client Onboarding Automation
    From referral to consent tracking to NDIS notification - with a live view of which forms are still outstanding. Automated follow-up reminders go out without you having to chase. No more mental tracking of who's sent what.
    Technical spec: Triggered by new referral (Astalty form submission or email detection by triage agent). Creates participant record in Astalty if not auto-created. Generates required consent forms from Astalty pre-populated templates (NDIS consent, AAC consent, service agreement, plan manager consent - plus any additional forms like housing or SA Health). Tracks signature status via Adobe Sign API. Dashboard view in Command Centre shows outstanding consents per participant. Automated follow-up reminders sent via email (Graph API) at 3-day and 7-day intervals. On completion: signed PDFs auto-uploaded to three locations (OneDrive, Astalty via bridge, SharePoint) and NDIS consent emailed to NDIS.
  • 3
    Invoice Reconciliation /invoice
    Astalty batch export, Xero, bank feed - done as a sequence, automatically. Bulk plan manager payments broken apart correctly without manual calculation. Your Sunday afternoon becomes a Sunday afternoon again.
    Technical spec: Weekly scheduled agent. Step 1: Pull completed Astalty tasks via data bridge, run batch invoice. Step 2: Auto-log each invoice to tax tracking spreadsheet (invoice number, plan manager, client initial, date, amount, 30% tax withholding). Step 3: Monitor Xero API bank feed for incoming payments. Step 4: Match bulk plan manager remittance payments against individual invoices using Xero API - break apart bulk payments automatically. Step 5: Flag unreconciled invoices after 14 days. Step 6: Generate weekly reconciliation summary to Sam via Telegram. Xero API (OAuth 2.0) for accounting integration, Astalty via Playwright bridge for task/invoice data.
  • 4
    Astalty Data Bridge
    Astalty has no API. We bridge it anyway. Your CRM connects to the rest of your stack without you being the bridge anymore. Data flows, reports pull correctly, and the systems that should talk to each other - finally do.
    Technical spec: Playwright browser automation running on Sam's machine. Logs into Astalty using stored credentials, navigates to required pages, extracts participant data, case notes, task records, and billing data via DOM scraping. Data cached locally in structured JSON. Scheduled sync runs every 30 minutes during business hours (8am-6pm ACST). Screenshots taken during extraction, data extracted, screenshots destroyed immediately for data compliance. Bridge exposes a local API endpoint that other agents (command centre, case notes, invoicing) query for Astalty data without needing their own browser sessions. All data stays on Sam's machine - never transmitted externally.
Phase 3

Scale the Team

3
Scale the Team
$6,500 indicative - Months 5-6, scheduled when you're ready
Goal: Extend the AIOS to the full team and catch the calls and compliance issues that currently fall through. Access Assist Co. runs on Sam's capacity. Phase 3 removes that ceiling.
  • 1
    Voice Agent for Phones
    AI answers when you can't. It identifies the caller, determines urgency, takes a message, and sends you a Telegram summary within seconds. You're never "Sam who doesn't answer the phone" again - because the phone is always answered.
    Technical spec: Twilio phone number provisioned (Australian +61) with call forwarding from Sam's business line when unanswered after 4 rings. Retell AI or ElevenLabs Conversational AI handles the call - identifies caller by name/number, asks reason for calling, determines urgency level, takes a structured message. Call summary delivered to Sam via Telegram Bot API within 30 seconds of call end, including caller name, urgency flag, reason, and audio recording link. Urgent calls (safeguarding, DCP, NDIS deadlines) flagged with priority notification. Integration with Microsoft Teams Phone if Sam completes that setup. Voice configured to sound natural and professional - not robotic. Tested with NDIS participant scenarios (vulnerable callers, distressed families).
  • 2
    Compliance Pre-Check Agent
    Automated weekly review of case notes against NDIS Practice Standards. Issues are flagged before they reach an auditor. You're audit-ready at any point in the year, not scrambling for three weeks before inspection.
    Technical spec: Weekly scheduled agent. Pulls all case notes from Astalty via data bridge. Claude reviews each note against NDIS Practice Standards compliance checklist: required fields present, correct charge item codes, appropriate documentation of participant consent, goal-linked service delivery, incident reporting compliance. Generates a compliance report flagging: missing information, formatting errors, notes that may not meet audit standards, overdue participant reviews. Report delivered to Sam via Telegram and added to Command Centre audit prep section. Tracks compliance score over time (percentage of notes passing all checks).
  • 3
    Team Command Centres
    Simplified dashboards for JC, Leah, and Christine. They see their participants, their tasks, their case notes. Not your full operational view. Each team member has what they need - and nothing they don't.
    Technical spec: Role-filtered views of the main Command Centre. JC view: documents to process, compliance uploads outstanding, onboarding consent tracking, admin task queue. Coordinator view (Leah/Christine/Monika): their assigned participants only, their tasks due, their case notes pending, their billable hours this week. No access to invoicing, revenue, KPI data, or other team members' participants. Accessed via unique URL per team member. Loom walkthrough videos created for each role. Christine's view prioritised for simplicity (minimal clicks, large text, clear labels).
  • 4
    Staff KPI Automation
    Cross-references timesheets against Astalty tasks. Flags when billable work is recorded as non-billable. Weekly report generated automatically. The Sunday night 10pm spreadsheet cross-referencing goes away permanently.
    Technical spec: Weekly scheduled agent. Pulls three data sources: Xero timesheets (hours per staff member), SharePoint detailed breakdown (day, billable/non-billable, client, minutes), and Astalty task records (logged billable work). Cross-references all three to identify: missing billable entries (work done but not logged in Astalty), incorrect categorisation (billable work logged as non-billable), timesheet vs Astalty hour discrepancies. Calculates per-staff KPIs: billable hours, billable percentage, revenue generated vs wage cost. Generates formatted weekly report to Sam via Telegram. Flags staff members below target billable percentage automatically.
